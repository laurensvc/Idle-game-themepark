# Design Context

## Users

Players of an idle/incremental theme park simulation game. They enjoy managing a virtual theme park, collecting rides, upgrading them for better income, and enjoying playful arcade-style gameplay. The experience should feel engaging, fun, and rewarding with clear progression indicators.

## Brand Personality

**Arcade • Playful • Retro-inspired • Vibrant**

The game embraces a bright, colorful arcade aesthetic with:
- **Emojis as icons** - Using emojis (🎠, 🎡, 🎢) to convey ride themes
- **Colorful palette** - Park green, orange, blue, red, yellow for different elements
- **Animation-heavy** - Floating particles, coin animations, shimmer effects
- **Gamified feedback** - Crit multipliers, golden tickets, visual pop effects

## Aesthetic Direction

**Visual Style**: Modern arcade/cabinet aesthetic with rounded corners, thick playful borders, vibrant gradients (yellow→orange), and shadow depth.

**References to capture**: 
- Retro arcade machine interfaces
- Modern incremental/idle games (e.g., Cookie Clicker but colorful)
- Nintendo Switch UI - bold borders, clear visual hierarchy

**Anti-references**: Dark/gritty industrial themes, minimalist corporate designs, serious/professional enterprise apps

**Theme**: The "arcade" theme is central to the entire design - from decorative dividers with gradient lines to animated backgrounds.

## Design Principles

1. **Arcade-first aesthetics**: Every visual element should feel like it belongs in a carnival/arcade setting - bold borders, vibrant colors, playful motion.

2. **Clear progression communication**: Income values, upgrade counts, and visitor metrics must be immediately readable with consistent formatting (tabular nums for money).

3. **Generous white space**: Cards use rounded corners with subtle shadows to create depth without clutter; empty states are calm but not boring.

4. **Icon-rich interface**: Emojis serve as primary icons - keep text minimal, let the visual identity speak through playful imagery.

5. **Rewarding micro-interactions**: Buttons scale on press, coins fly off-screen with physics-based animations, feedback pulses visually (crit numbers float upward).
